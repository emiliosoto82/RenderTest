# M00 – Baseline

Este proyecto ya existe.
No es un ejercicio vacío ni un ejemplo de clase.

La API **funciona**, compila y responde correctamente.
Antes de cambiar nada, tu trabajo es **entender qué tienes delante**.

Este ejercicio no va de añadir cosas todavía.
Va de **saber desde dónde partes**.

---

## Objetivo

Asegurarte de que entiendes el estado actual del proyecto **antes** de que empiece el caos.

Cuando esta misión termina, deberías poder afirmar con seguridad:

- ✅ El proyecto arranca
- ✅ Los endpoints funcionan
- ✅ Sabes qué archivos existen y para qué sirven
- ✅ Sabes cuál es el estado “sano” del sistema

Este estado se considera el **baseline**.

---

## Lo que debes hacer

1. Clona el repositorio
2. Abre el proyecto en tu IDE
3. Arranca la aplicación
4. Comprueba que:
   - `/health` responde correctamente
   - `/status` responde correctamente
5. Recorre el proyecto:
   - Estructura de carpetas
   - Archivos principales
   - Modelos
   - Configuración

**No modifiques nada todavía.**

---

## Reglas importantes

- ❌ No cambies código
- ❌ No crees ramas
- ❌ No hagas commits
- ❌ No “mejores” nada
- ✅ Solo observa y entiende

Si algo te parece raro:
- anótalo mentalmente
- **no actúes todavía**

---

## Qué No es esta misión

- No es para aprender Git
- No es para refactorizar
- No es para “arreglar cosas”
- No es para demostrar que sabes más

Todo eso vendrá después.

---

## Entrega

Cuando hayas terminado:

- El repositorio debe estar **idéntico** a como estaba al comenzar
- `git status` debería mostrar un estado limpio
- Debes ser capaz de explicar:
  - qué hace la API
  - qué endpoints existen
  - qué datos maneja

No se entrega ningún código.
Solo continúas a la siguiente misión.

---

## Aviso final

A partir de la siguiente misión:
- El sistema empezará a cambiar
- Aparecerán errores
- Habrá decisiones contradictorias
- El repositorio dejará de ser limpio

Este es el **último momento de calma**.

Memoriza cómo funciona ahora.
Después, lo necesitarás.
